Add Functionality

Events and actions

Explain all functionality is delivered through events and actions...

Decision tree - go through user story, for each step:

  • what will the user see? - when navigate to page, after any actions
  • what will the user interact with? - select element/page, event
  • what will happen when this event occurs? - add action
  • should include page navigation - link to that from create page section
  • may include running api requests or using api data - link to api section

Connect to External Data

Add an API to your project

Explain auth etc...

Decision tree for each route:

  • where do you want to use the data from this endpoint? - add to relevant page
  • when do you want the request ro run - add run request action to relevant event
  • where will the parameters come from etc
  • where will this response field be used, is it in an array, is it an image etc
  • do you want anything to happen when the response is received (status code events)

NOTES

  • Articles modular and discoverable by search/links
    • each one works as a standalone unit but is extensively linked - encourages ideal workflow but supports variety of pathways
  • Main content will be focused on instructional material, with reference material not shown by default (e.g. using tooltip / info button)
    • reference material would be e.g. canvas specs, list of to do errors, list of elements/actions, name validation, data types, page config, saving
  • Equivalent of quick tip material (use cases implemented in workbench) to be included, ideally within this modular article structure
  • Any info included in interactive format to be duplicated in linear format for different learners
  • Videos to include bite-sized intros (simple use cases) for help drawer and docs, plus full walkthroughs to accompany tutorials
    • as standalone chapters but also stitched together as courses
    • help drawer will comprise list of videos
  • Additional formats/media to include animated gifs (for short sequences of steps), more show/hide content e.g. tabbed layout as alternative to decision trees where more content is required
